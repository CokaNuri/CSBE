# 02 인공지능 소프트웨어 품질

전통적인 소프트웨어와의 차이점

> 데이터 학습을 통해 기능을 수행하는 시스템이므로
> 어떤 데이터를 학습하느냐에 따라 시스템의 출력이 달라질 수 있다.

기계학습 시스템

> 학습 알고리즘 혹은 학습 모델을 따르는 하나 이상의 소프트웨어 컴포넌트를 포함한 학습 기반 시스템

## 인공지능 소프트웨어의 품질 특성

| 요소             | 설명                                                             |
| ---------------- | ---------------------------------------------------------------- |
| 투명성           | 입력에 대한 결과 값 재현                                         |
| 책임             | 출력 해석 가능성                                                 |
| 다양성,공정성    | 특정 요소에 편중하여 출력 제공하면 안됨                          |
| 보안과 안전성    | 데이터가 좋은 품질과 무결성을 가져야함 노이즈,악의적 데이터 대처 |
| 견고성과 신뢰성  | 유해한 입력에 대해서도 믿을 만한 결과 제공                       |
| 법적·윤리적 측면 | 법적·윤리적 문제 보장                                            |

## 시스템 관점별 품질 요소

> 기계학습 기반 소프트웨어 시스템 구성

![alt text](image.png)

| 요소        | 설명                                                          |
| ----------- | ------------------------------------------------------------- |
| 모델 관점   | 학습 모델과 관련, 데이터에 대한 훈련                          |
| 데이터 관점 | 모델에 입력되는 데이터와 관련된 부분                          |
| 시스템 관점 | 모델과 데이터들을 연결하고 형상을 정의                        |
| 인프라 관점 | 하드웨어 혹은 라이브러리 등을 이용해 어떻게 구현되는가에 초점 |
| 환경 관점   | 시스템과 사용자가 어떻게 상호작용하는가를 표현                |

### 모델 관점 품질 요소

| 요소        | 설명                                               |
| ----------- | -------------------------------------------------- |
| 모델 타당성 | 모델 유형이 적합한가                               |
| 적합성      | 기능이 데이터를 기반으로 정확하게 수행되는가       |
| 견고성      | 누락 혹은 오류 있는 데이터를 잘 처리할 수 있는가   |
| 안정성      | 서로 다른 데이터에 대해 반복적인 결과를 생성하는가 |
| 공정성      | 출력이 공정한 결정을 제시하는가                    |
| 해석능력    | 훈련된 모델의 내용을 사람이 해석할 수 있는가       |

### 데이터 관점 품질 요소

| 요소   | 설명                                                       |
| ------ | ---------------------------------------------------------- |
| 대표성 | 데이터가 모집단을 대표하는가                               |
| 정확성 | 데이터가 오류 없이 개발되었는가                            |
| 완전성 | 누락된 데이터가 없는가                                     |
| 유통성 | 데이터가 최신 내용을 포함하는가                            |
| 독립성 | 훈련 데이터와 테스트 데이터가 상호 독립적인가              |
| 일관성 | 서로 다른 데이터셋에 대해 형식,표본 추출이 일관성이 있는가 |

### 환경 관점의 품질 요소

| 요소      | 설명                                             |
| --------- | ------------------------------------------------ |
| 환경 영향 | 훈련 과정이 환경에 영향을 미치는 정도            |
| 사회 영향 | ML 컴포넌트가 사회에 영향을 미치는 정도          |
| 범위 준수 | ML 컴포넌트가 의도된 사용범위 내에 존재하는 정도 |

### 인프라 관점의 품질 요소

| 요소          | 설명                                               |
| ------------- | -------------------------------------------------- |
| 인프라 적합성 | 인프라 구성 요소가 ML 컴포넌트의 요구를 충족하는가 |
| 훈련 효율성   | 학습 모델을 훈련하기 위한 자원이 유용한가          |
| 실행 효율성   | 훈련된 모델을 실행시키기 위한 자원이 유용한가      |

### 시스템 관점의 품질 요소

| 요소        | 설명                                                     |
| ----------- | -------------------------------------------------------- |
| 효과성      | 출력 제어 알고리즘이 오류 출력을 잘 탐지하는가           |
| 제어 효율성 | ML 컴포넌트를 모니터링하기 위해 사용되는 자원이 유용한가 |
| 효과성      | 범주 제어 알고리즘이 문맥 변경을 탐지하는 능력           |
| 제어 효율성 | 응용 범주를 모니터링하기 위해 사용되는 자원이 유용한가   |
