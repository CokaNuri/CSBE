    04 소프트웨어 프로세스 개선¶

    소프트웨어 프로세스를 개선하기 위한 가이드라인 및 표준

    CMM과 CMMI¶

    CMM¶

    시스템 개발 프로세스의 성숙도 모델

    • 성숙도: 개발 활동을 얼마나 잘하고 있는지 나타내는 척도
    • 미 국방부가 방산 업체의 프로세스 신뢰도를 가늠하려고 출발시킨 모델
    • 단점: 분야별 모델이 난립해 통합 적용이 어려움

    CMMI¶

    흩어진 CMM들을 하나로 통합한 모델 프로세스 영역 + 능력도로 표현

    구성 요소 예
    프로세스 영역 프로젝트 계획, 리스크 관리, 형상 관리, 품질 보증
    능력도(수준) 수준 1 ~ 수준 5

    능력도 5단계는 아래로 갈수록 사람이 바뀌어도 일정한 품질이 나오는 조직을 뜻함

    수준 이름 상태
    수준 1 수행 초기 소프트웨어 개발
    수준 2 관리 기본적인 프로젝트 관리 가능
    수준 3 정의 조직 표준 프로세스로 표준화
    수준 4 정량적 관리 측정 데이터로 프로세스를 정량적으로 관리
    수준 5 최적화 데이터 기반으로 지속적으로 개선
    수준 1부터 수준 5까지 각각 어떤 차이가 있는가? 예측 가능성과 통제력이 올라간다 1은 매번 다름 → 2는 프로젝트별 반복 가능 → 3은 조직 표준화 → 4는 수치로 예측 → 5는 데이터로 자기개선
    데이터로 정량 관리에서 그 데이터란 무엇인가? 프로세스와 산출물을 측정한 정량 지표 대표적인 예: - 생산성: 투입 시간당 처리한 코드량 - 일정·비용 편차: 계획 대비 실제 차이 - 테스트 커버리지: 전체 코드 중 테스트가 실행한 비율 - 평균 결함 수정 시간: 결함 하나를 고치는 데 걸린 평균 시간 수준 4는 이런 지표를 통계적으로 관리해 결과가 예측 가능한 범위 안에 들도록 함 수준 5는 그 데이터를 근거로 프로세스 자체를 더 좋게 바꿈
    CMMI 2.0은 무엇이 다른가? 초점이 프로세스를 따랐는가에서 사업 성과가 좋아졌는가로 옮겨감 - Process Area라는 말 대신 Practice Area를 씀 - 중복 실천을 줄여 모델이 더 가벼워짐 - 애자일 방식을 쓰는 조직을 위한 지침이 추가

    SPICE & A-SPICE¶

    단계적인 프로세스 성숙도를 정의하고 ISO 표준으로 제정되어 활용됨

    SPICE¶

    프로세스 생명주기 기반, 레벨 0 ~ 레벨 5

    • CMMI와 발상은 비슷하나, 특정 기관 모델이 아니라 ISO 국제표준이라는 점이 다름

    A-SPICE¶

    SPICE 표준을 자동차 산업에 맞게 수정한 것

    • 레벨 0 ~ 레벨 5로 구분
    • 핵심 / 조직 / 자원 수명주기라는 3개의 수명주기 모델 영역에 대해 8개의 프로세스 그룹 존재
    • 독일 자동차협회 주도, 완성차 회사가 부품·임베디드 SW 협력업체의 개발 프로세스를 평가할 때 사용

    식스 시그마 (Six Sigma)¶

    결함의 원인을 식별·제거해 공학적 활동의 품질을 개선하는 것이 목표. 99.99966% 수준까지 결함을 줄이는 것을 지향

    DMAIC과 DMADV를 기반으로 함

    방법론 단계 언제 쓰나
    DMAIC Define → Measure → Analyze → Improve → Control 이미 있는 프로세스를 개선할 때
    DMADV Define → Measure → Analyze → Design → Verify 새 프로세스·제품을 처음 설계할 때

    기타 프로세스 표준¶

    소프트웨어/시스템을 만들고 운영하는 동안 어떤 프로세스들이 있어야 하는지를 정의한 참조 목록

    표준 범위 특징
    ISO/IEC 12207 소프트웨어 생명주기 SW 프로세스의 표준 뼈대
    ISO/IEC 15288 시스템 생명주기 SW를 넘어 하드웨어·사람까지 포함한 시스템 전체 버전
    ISO/IEC 29110 초소형 조직 소규모 팀(~25명)을 위함

    12207은 소프트웨어에 한정, 15288은 그 SW가 들어가는 시스템 전체를 다룸

    🔑 최종 정리¶

    소프트웨어 프로세스 개선의 출발점은 일을 얼마나 잘하고 있나를 객관적으로 재는것 그래서 조직의 성숙도를 5단계로 매기는 CMM이 나왔고, 분야별로 난립한 CMM을 하나로 합친 게 CMMI 국제표준으로 통일한 게 SPICE이고, 그걸 자동차 산업에 특화한 게 A-SPICE 식스 시그마는 통계적으로 결함을 줄이는 또 다른 개선 접근
    프로세스 표준 이라는 기준을 이용해 프로세스 수준을 정한다.

    한 줄 요약: 프로세스 표준이 기준을 주고, 성숙도·역량 모델이 그 기준으로 조직을 평가하고 개선의 방향을 잡는다
