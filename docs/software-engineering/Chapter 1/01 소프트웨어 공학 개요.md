# 소프트웨어 고장 사례

> 핵심 : 소프트웨어 결함은 돈·생명까지 앗아간다 <br>
> → 그래서 소프트웨어 공학이 필요하다 각 사례를 "어떤 품질이 무너졌나"로 읽어보자

## 화성 기후 관측 위성 (1999)

- 원인: **단위 불일치**. 국제 표준 단위인 1(N·s) 로 표시해야 하는데 미국식인 4.45(lb·s)로 표시
- 결과: MCO 위성이 사라짐 -> 큰 경제적 손실
- 교훈: 표준을 잘 따르자. 문서화를 잘 하자.

## 자동차 해킹

- 원격으로 차량 제어 시스템에 침입해 핸들·브레이크·엔진 조작 가능 (예: 2015 짚 체로키)
- 교훈: **보안성**이 곧 **안전성**과 직결. 차가 인터넷에 연결되며 보안 결함이 인명 위협으로

## 보잉 737 MAX8 추락

- 원인: **MCAS**라는 SW기반 제어 시스템에 입력되는 노즈 센서 값 오류<br>
  하나의 노즈 센서 값만 이용하여 동작하도록 개발
- 결과: 두 사고로 346명 사망, 전 세계 737 MAX 약 20개월 운항 정지.
- 교훈: **단일 장애점**. 최소 두 센서를 교차 확인했어야 함. 견고성·안전성 설계 실패

## 테슬라 자율주행

- 오토파일럿/FSD 사고. 운전자 과신 + AI가 특정 상황 오인
- 결과 : 테슬라 모델 S 오토파일럿 때문에 운전자 사망 사고
- 교훈: AI 견고성, 보조 시스템을 최종 결정에 쓰면 안 된다는 범위 준수

## 기타 소프트웨어 사고 일지

- Therac-25(방사선 치료기 SW 결함 → 환자 사망)
- Ariane 5(숫자 변환 오버플로로 로켓 폭발)
- 산업, 과학기술 분야에서 SW 결함으로 인한 사고 발생

# 소프트웨어 위기

## 원인과 증상

- 원인: **복잡성 증가** + **소프트웨어 공학의 상대적 미성숙**
- 증상: 예산·일정 초과, 낮은 품질, 유지보수 곤란

## 소프트웨어 개발이 어려운 이유 = 소프트웨어의 특성

- **비가시성**: 눈에 안 보여 진행·구조 파악이 어려움
- **복잡성**: 상태·경로가 폭발적으로 많음
- **순응성**: 하드웨어·규정·요구에 계속 맞춰 변경됨
- **변경 용이성의 함정**: 고치기 쉬워 보여 무분별하게 바뀌고 더 망가짐
- 복제는 공짜지만 개발은 비쌈

## 인공지능 시대의 소프트웨어 위기

- 전통적 위기 + 데이터 의존·불투명성·편향·예측 불가능성
- "코드를 잘 짜는 것"만으로 품질이 보장되지 않는 시대

# 소프트웨어 공학 기술의 적용

> 위기의 답으로 등장한 것이 소프트웨어 공학

## 소프트웨어 공학적 기법의 종류

- **구조적 프로그래밍**: 순차·선택·반복 세 구조로만 짜서 이해·검증 쉽게
- **객체지향 프로그래밍**: 데이터+기능을 객체로 묶어 재사용성·유지보수성 ↑
- **소프트웨어 컴포넌트 및 재사용**: 검증된 부품을 조립해 품질·생산성 ↑
- **개발 프로세스**: 폭포수·애자일 등 체계적 진행 틀 (프로세스 품질과 연결)
- **검사 및 검증(V&V)**: 제대로 만들었는지 확인
- **형상 관리**: 버전·변경 이력 통제
- **소프트웨어 아키텍처**: 시스템의 큰 구조·골격 설계

## 소프트웨어 공학의 정의와 원리

- 정의: 소프트웨어를 **체계적·규율적·정량적으로** 개발·운영·유지보수하는 공학적 접근

### 원리

| 원리          | 설명                                  |
| ------------- | ------------------------------------- |
| 엄격성·정형성 | 모호함 없이 정확하게 기술             |
| 관심사 분리   | 문제를 독립적 측면으로 나눠 따로 다룸 |
| 모듈화        | 시스템을 독립적 모듈로 분할           |
| 추상화        | 핵심만 남기고 세부는 감춤             |
| 변경 예측     | 미래의 변경을 미리 고려해 설계        |
| 일반화        | 더 넓게 재사용 가능한 형태로          |
| 점진성        | 조금씩 단계적으로 발전시킴            |
| 명세화        | 무엇을 만들지 명확히 정의·문서화      |
